# 微信公众号精美排版技能

将用户提供的文本内容，转化为专业、精美、可直接粘贴至微信公众号编辑器的 HTML 排版，并输出为以文章标题命名的 HTML 文件。

---

## ⚠️ 铁律：绝对不得改动原文

> 排版的职责只有一个：改变文字的**呈现形式**，而不是**内容**。
>
> - ❌ 禁止修改、删减、概括、改写任何一句原文
> - ❌ 禁止"优化措辞"、"补充说明"、"省略重复"
> - ❌ 禁止把原文列表合并为段落，或把段落拆解重组
> - ✅ 可以重新分组（把相关内容放进同一张卡片）
> - ✅ 可以提取为视觉组件（数字→数据卡片，步骤→编号列表）
> - ✅ 前提：**每一个字都完整保留**

---

## 核心工作流

1. **阅读参考文件**：先读 `references/wechat-constraints.md`，再读对应风格的示例片段（`references/styles/`）
2. **识别内容类型，自动匹配风格**（见"风格自动识别"）
3. **判断文章长度**，选用对应的结构策略（见"长度适配"）
4. **检查用户是否声明了品牌色**，若有则替换风格主色
5. **排版生成**：遵守 CSS 约束，参考风格示例
6. **提取或生成标题**，保存 HTML 文件，呈现给用户

生成前，用 1-2 句话告知用户：选了什么风格、主色调、排版思路。

## 风格自动识别

**不要让用户选风格。** 自动判断后告知用户选了什么及原因。

### 决策树

```

  ├─ 行业/财经/企业语境 → 【商务精英】
  └─ 产品/技术/代码语境 → 【科技极客】 或 【暗夜极客】（后者更适合极客/硬核受众）

内容有强烈情感/故事/生活感悟？

内容有浓厚中国传统文化元素（古诗/历史/节气/国潮）？
  → 【国风雅韵】

内容是职场/成长/思考/管理/自我提升类？
```

## 文章长度适配

根据字数选择对应的结构策略：

| 长度 | 字数参考 | 结构策略 |
|------|----------|----------|
| **短文** | < 600 字 | 不强加卡片和小标题；1-2 个视觉强化元素即可；结构保持扁平 |
| **中等** | 600–2000 字 | 2-4 个章节小标题；适量卡片和引用块；正常使用分割线 |
| **长文** | > 2000 字 | 文章开头加导读摘要卡片；每个章节有明确视觉锚点；章节间用分割线区隔；避免长段大块纯文字 |

## 品牌色支持

如果用户在对话中声明了品牌色（如"我们的主色是 `#1677ff`"），或上传了品牌手册：

1. 用品牌主色替换风格中的"主色调"（标题色、左边框色、高亮色）
2. 衍生配套色：主色加白 80% 作为背景卡片色，主色加深 20% 作为强调色
3. 其余风格特征（圆角大小、字体规范、结构模式）保持不变
4. 在风格说明中注明："已应用您的品牌色 [颜色值]"

## CSS 兼容性约束（速查）

详细规则见 `references/wechat-constraints.md`，每次生成前必须重新阅读。核心禁止项：

- ❌ `display: flex / grid / inline-block`（粘贴后被剥离或失效）
- ❌ CSS 变量 `var()`、`calc()`、`@media`、`@keyframes`、伪元素
- ❌ `position: fixed/absolute/sticky`、`float`、`overflow: hidden`
- ❌ 外链字体、CSS 缩写属性（用展开写法）、`<style>` 标签块
- ⚠️ `<div>` 上的背景色和边框不稳定 → 重要视觉元素改用 `<table>`
- ⚠️ `box-shadow` 在 Android 部分版本忽略 → 不作为主要视觉元素
- ✅ 带 `border-radius` 的 `<table>` 必须加 `border-collapse: separate; border-spacing: 0;`
- ✅ 所有 `<table>` 必须加 `border="0" cellpadding="0" cellspacing="0"` + 所有 `<td>` 加 `border: none;`
- ✅ 块级元素宽度**不得使用超过 350px 的固定像素值**，一律改用百分比（`width: 90%`）或省略

## 配图处理

1. **占位图**：文案没有提供图片但排版需要时，插入：
   `https://iph.href.lu/[宽]x[高]?text=[建议配图描述]`
2. **本地图片**：正常渲染本地路径供预览；"复制到公众号"按钮会自动将本地路径替换为占位图

## 输出：文件命名与保存

1. 从原文提取标题；若无明确标题，根据内容生成简洁中文标题（不超过 15 字）
2. 文件名：`{标题}.html`（去除特殊字符，空格替换为下划线）
3. 保存路径：**当前工作目录**（`./`），不硬编码任何绝对路径
4. 生成后：
   - 有 `present_files` 工具 → 调用呈现给用户
   - 没有 → 告知用户文件名和保存位置

预览外壳结构见 `references/template.html`，直接使用该模板，不要在 SKILL.md 中重复粘贴。

## 排版质量检查清单

- [ ] 原文每一个字都完整保留？（铁律）
- [ ] 整体视觉有清晰的"一张脸"，没有风格杂烩？
- [ ] 文章长度与结构策略匹配？（短文不强加卡片，长文有导读）
- [ ] 如用户有品牌色，已正确应用？
- [ ] `#article-content` 内无 flex/grid/inline-block/CSS变量/伪元素？
- [ ] 块级元素无超过 350px 的固定像素宽度？（用百分比替代）
- [ ] 所有 `<table>` 有 `border="0" cellpadding="0" cellspacing="0"`？
- [ ] 所有 `<td>` 有 `border: none;`（或显式边框）？
- [ ] 文件已以文章标题命名，保存到当前工作目录？
